# InkMuse 运行文档

本文档说明如何在本地运行 InkMuse（父仓库 + `backend` + `frontend`）。

## 1. 环境要求

- Git
- Go `1.24+`（后端 `go.mod` 为 `go 1.24.0`）
- Node.js `20+` 与 npm
- Docker 与 Docker Compose（用于 PostgreSQL）

## 2. 拉取代码（含子模块）

首次拉取建议使用：

```bash
git clone --recurse-submodules https://github.com/nsxzhou/inkmuse.git
cd inkmuse
```

如果已经克隆过但子模块未初始化：

```bash
git submodule update --init --recursive
```

## 3. 启动后端（默认 PostgreSQL）

在第一个终端执行：

```bash
cd backend
cp configs/config.yaml.example configs/config.yaml
cp .env.example .env
docker compose up -d postgres
# 编辑 backend/.env，把 INKMUSE_LLM_API_KEY 改成你的真实 key
./scripts/run-local.sh
```

说明：

- `./scripts/run-local.sh` 会在检测到 `backend/.env` 时自动加载该文件。
- `./scripts/run-local.sh` 会在 `provider=postgres` 时自动执行迁移，再启动服务。
- 后端默认监听 `http://127.0.0.1:8080`。
- `INKMUSE_LLM_API_KEY` 必须是非空值；未设置真实 key 时，LLM 相关接口会在调用时失败。

可选健康检查：

```bash
curl -sS http://127.0.0.1:8080/healthz
curl -sS http://127.0.0.1:8080/readyz
```

## 4. 启动前端

在第二个终端执行：

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

说明：

- 前端默认地址：`http://127.0.0.1:5173`
- `.env.example` 默认配置：
  - `VITE_API_BASE_URL=http://127.0.0.1:8080/api/v1`

浏览器访问：

```text
http://127.0.0.1:5173
```

## 5. 停止服务

- 停止前后端：在各自终端按 `Ctrl+C`
- 停止 PostgreSQL：

```bash
cd backend
docker compose down
```

如果需要同时删除数据库卷数据：

```bash
cd backend
docker compose down -v
```

## 6. 常见问题

1. 子模块目录为空或缺文件

```bash
git submodule update --init --recursive
```

2. 后端启动报环境变量错误（如 `INKMUSE_DATABASE_URL` / `INKMUSE_LLM_API_KEY`）

- 确认 `backend/.env` 存在且变量值非空（尤其是 `INKMUSE_LLM_API_KEY`）
- 修改 `.env` 后，重新执行 `backend/scripts/run-local.sh`

3. 端口占用（`5432`、`8080`、`5173`）

- 关闭占用进程，或按需修改对应配置/启动参数
