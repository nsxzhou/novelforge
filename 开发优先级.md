# NovelForge 后端 V1 开发优先级

## 背景与范围

当前仓库后端已完成基础设施骨架与 V1 领域模型/存储边界：具备 Hertz 服务启动、配置加载与校验、基础中间件、健康检查路由、LLM 占位工厂、全部 6 个领域聚合的实体定义与仓储接口、内存存储实现，以及服务用例接口声明。尚未实现业务服务、业务 API 和 LLM 真实客户端。

V1 开发优先围绕最小闭环推进：

`灵感 -> 项目创建 -> 设定/大纲 -> 单章生成 -> 当前稿确认`

> 本文档仅用于定义开发优先级，不包含具体业务功能实现。

## 当前代码基线

- 启动装配已存在：`backend/internal/app/bootstrap.go`
- HTTP 服务与中间件注册已存在：`backend/internal/infra/http/server.go`
- 路由目前仅健康检查与占位接口：`backend/api/http/routes.go`
- 配置模型已扩展 `StorageConfig`：`backend/pkg/config/config.go`
- LLM 仅占位客户端：`backend/internal/infra/llm/client.go`、`backend/internal/infra/llm/factory.go`
- **领域模型已落地**：`backend/internal/domain/{project,asset,chapter,conversation,generation,metric}/`
- **存储层已实现**：`backend/internal/infra/storage/`（含 `memory/*` 内存仓储及测试）
- **服务用例接口已定义**：`backend/internal/service/*/service.go`（仅接口，无实现）

## 推荐开发优先级（V1）

### 1. ✅ 先落地领域模型与存储边界（最高优先）—— 已完成

- 目标：定义并落地 V1 必需对象（Project / Asset / Chapter / Conversation / GenerationRecord / MetricEvent）及仓储接口。
- 原因：没有稳定数据模型，后续 API、生成链路与"当前稿确认"都无法稳定推进。
- **完成情况**：
  - 6 个领域聚合各含 `model.go`、`repository.go`、`model_test.go`
  - 存储层：`Repositories` 组合 + `NewRepositories` 工厂 + 内存仓储实现与测试
  - 服务层：6 个 `UseCase` 接口与 `Dependencies` 声明已定义
  - 配置层：新增 `StorageConfig`（含 `provider` 字段与校验）
- 关键文件：
  - 已完成：`backend/internal/domain/{project,asset,chapter,conversation,generation,metric}/`
  - 已完成：`backend/internal/infra/storage/` 及 `storage/memory/`
  - 已完成：`backend/internal/service/*/service.go`
  - 已更新：`backend/pkg/config/config.go`

### 2. ✅ 实现"项目创建 + 设定资产管理"基础 API —— 已完成

- 目标：完成对话开题最小能力（书名、简介），并支持世界观/角色/大纲的保存与查询。
- 原因：这是 V1 闭环的前半段，且是章节生成的输入前置。
- **完成情况**：
  - 已完成 Project API：创建 / 列表 / 查询 / 更新
  - 已完成 Asset API：创建 / 列表 / 按类型过滤 / 查询 / 更新 / 删除
  - 已完成 `service/project`、`service/asset` 用例实现，并接入统一错误映射
  - 已完成 PostgreSQL 运行时接入：配置切换、存储工厂、bootstrap 注入、migration 入口
  - 已补齐 handler 集成测试与 PostgreSQL repository 测试，并完成一次显式 migration + 启动服务的运行态链路验证
- 关键文件：
  - 已更新：`backend/api/http/routes.go`
  - 已新增：`backend/internal/infra/http/handler/project.go`
  - 已新增：`backend/internal/infra/http/handler/asset.go`
  - 已新增：`backend/internal/service/project/**`、`backend/internal/service/asset/**`
  - 已补充测试：`backend/internal/infra/http/handler/handler_integration_test.go`
  - 已补充测试：`backend/internal/infra/storage/postgres/repository_test.go`
  - 已确认 PG 接入链路：`backend/internal/infra/storage/factory.go`、`backend/internal/app/bootstrap.go`、`backend/internal/infra/storage/migrate.go`、`backend/cmd/migrate/main.go`

### 3. 接入可用的 OpenAI 兼容 LLM 客户端 + Prompt 模板配置

- 目标：把当前占位 `LLMClient` 替换为可调用实现，并支持模板可配置。
- 原因：V1 的生成能力（设定生成、章节生成）依赖真实模型调用。
- 关键文件：
  - 更新：`backend/internal/infra/llm/factory.go`
  - 更新：`backend/internal/infra/llm/client.go`
  - 更新：`backend/pkg/config/config.go`
  - 更新：`backend/configs/config.yaml.example`
  - 可新增：`backend/internal/infra/llm/prompts/**`

### 4. 实现"对话驱动微调"服务层（对话为主，手动编辑为辅）

- 目标：支持对话指令修改 Project/Asset；同时保留手动编辑接口用于局部修正。
- 原因：这是产品原则中的双轨微调核心，不做会导致用户只能一次性生成无法迭代。
- 关键文件：
  - 新增：`backend/internal/service/conversation/**`
  - 新增：`backend/internal/infra/http/handler/conversation.go`
  - 更新：`backend/api/http/routes.go`

### 5. 实现章节生成主链路（单章生成 / 续写 / 局部重写）

- 目标：基于大纲和资产上下文生成章节草稿，并记录 GenerationRecord（token、耗时、状态）。
- 原因：这是 V1 价值点"稳定出稿"的核心能力。
- 关键文件：
  - 新增：`backend/internal/service/chapter/**`
  - 新增：`backend/internal/infra/http/handler/chapter.go`
  - 新增：`backend/internal/domain/chapter/**`
  - 新增：`backend/internal/domain/generation/**`
  - 更新：`backend/api/http/routes.go`

### 6. 实现"当前稿确认"状态流转

- 目标：支持章节草稿确认，维护 `current_draft_id`、`current_draft_confirmed_at` 等字段。
- 原因：这是 V1 闭环终点，也是 KPI（草稿采纳率）的统计基础。
- 关键文件：
  - 更新：`backend/internal/domain/chapter/**`
  - 更新：`backend/internal/service/chapter/**`
  - 更新：`backend/internal/infra/http/handler/chapter.go`

### 7. 埋点与可观测性（V1 先采集，不做看板）

- 目标：按规划采集调用次数、失败率、耗时、token 成本相关事件。
- 原因：V1.1 成本看板依赖这些基础数据，且有助于快速发现生成链路问题。
- 关键文件：
  - 新增：`backend/internal/domain/metric/**`
  - 新增：`backend/internal/service/metrics/**`
  - 在生成/确认流程中接入埋点调用

### 8. 补齐测试与回归基线（每个里程碑同步完成）

- 目标：为配置校验、服务用例、HTTP handler 和关键流程增加测试，避免后续迭代不稳定。
- 关键文件：
  - 新增：`backend/pkg/config/config_test.go`
  - 新增：`backend/internal/service/**/*_test.go`
  - 新增：`backend/internal/infra/http/handler/**/*_test.go`
