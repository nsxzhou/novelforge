# NovelForge 后端 V1 开发优先级

## 背景与范围

当前仓库后端已完成基础设施骨架、V1 领域模型/存储边界、OpenAI 兼容 LLM 客户端与 Prompt 模板基础设施，以及 Project / Asset 的对话驱动微调闭环与章节生成主链路：具备 Hertz 服务启动、配置加载与校验、基础中间件、健康检查路由、全部 6 个领域聚合的实体定义与仓储接口、内存 / PostgreSQL 存储实现、项目与资产基础 API、对话建议生成 / 显式确认写回、章节生成 / 当前稿确认 / 续写 / 局部重写与 GenerationRecord 持久化。指标采集链路已接入 chapter/conversation 主流程，但当前仍未包含看板与真实 token 成本口径。

V1 开发优先围绕最小闭环推进：

`灵感 -> 项目创建 -> 设定/大纲 -> 对话微调 -> 单章生成 -> 当前稿确认`

> 本文档仅用于定义开发优先级，不包含具体业务功能实现。
