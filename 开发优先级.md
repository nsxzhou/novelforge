# NovelForge 后端 V1 开发优先级

## 背景与范围

当前仓库后端已完成基础设施骨架、V1 领域模型/存储边界、OpenAI 兼容 LLM 客户端与 Prompt 模板基础设施，以及 Project / Asset 的对话驱动微调闭环：具备 Hertz 服务启动、配置加载与校验、基础中间件、健康检查路由、全部 6 个领域聚合的实体定义与仓储接口、内存 / PostgreSQL 存储实现、项目与资产基础 API、对话建议生成 / 显式确认写回，以及对应服务用例实现。当前尚未实现章节生成、当前稿确认与指标采集等后续业务链路。

V1 开发优先围绕最小闭环推进：

`灵感 -> 项目创建 -> 设定/大纲 -> 对话微调 -> 单章生成 -> 当前稿确认`

> 本文档仅用于定义开发优先级，不包含具体业务功能实现。

## 当前代码基线

- 启动装配已存在：`backend/internal/app/bootstrap.go`
- HTTP 服务与中间件注册已存在：`backend/internal/infra/http/server.go`
- 路由已接入健康检查、Project / Asset CRUD 与 Conversation refinement API：`backend/api/http/routes.go`
- 配置模型已扩展 `StorageConfig`，并纳入 refinement prompt 配置校验：`backend/pkg/config/config.go`
- LLM 基础设施已接入 OpenAI 兼容客户端、Prompt 模板注册表与模板渲染能力：`backend/internal/infra/llm/factory.go`、`backend/internal/infra/llm/openai_client.go`、`backend/internal/infra/llm/prompts/**`
- **领域模型已落地**：`backend/internal/domain/{project,asset,chapter,conversation,generation,metric}/`
- **存储层已实现**：`backend/internal/infra/storage/`（含 `memory/*`、`postgres/*` 及 conversation `pending_suggestion` 持久化）
- **服务用例已部分实现**：`backend/internal/service/project/**`、`backend/internal/service/asset/**`、`backend/internal/service/conversation/**` 已完成；`backend/internal/service/{chapter,generation,metric}/` 仍以接口与依赖声明为主

## 推荐开发优先级（V1）

- 目标：定义并落地 V1 必需对象（Project / Asset / Chapter / Conversation / GenerationRecord / MetricEvent）及仓储接口。
- 原因：没有稳定数据模型，后续 API、生成链路与"当前稿确认"都无法稳定推进。
- **完成情况**：
  - 6 个领域聚合各含 `model.go`、`repository.go`、`model_test.go`

- 关键文件：
  - 已完成：`backend/internal/domain/{project,asset,chapter,conversation,generation,metric}/`
